# 互动与态度挖掘（Interaction）

> 对应「这个人怎么说话、怎么回应、怎么处理冲突」。
> 适用角色：所有角色均需要，但维度侧重因角色而异。
> 设计参考：persona-based MoE 的行为分路、DPRF 的可观测行为框架。

## 原则

- **互动层不是性格测评**：目标是捕捉「什么条件下怎么反应」的模式。
- **原话摘录优先**：短语级摘录即可，避免整段搬运。
- **用户印象单独标注**：永远不混进被模拟者的行为证据区。

## 要抽取的维度

### 1. 默认沟通方式
回复长度偏好、结构偏好（列表/段落）、渠道偏好（文字/语音/截图）、响应速度特征。

### 2. 提问与推回
什么条件下要求补充信息？推回时的典型话术？推回方式（直说/反问/替代方案）？

### 3. 质疑偏好
更爱挑战什么？质疑语气如何？有没有固定的质疑句式？

### 4. 冲突处理
对含糊需求、文档缺失、跨团队扯皮时的典型应对。

### 5. 排斥场景（互动层 Gotchas）
绝不做的沟通类型——不接语音、不周末 @、不在群里讨论敏感话题等。

### 6. 情感互动（非职场角色新增）

仅当角色为亲人/伴侣/朋友时需要：
- 表达关心/爱意的方式
- 撒娇/示弱/逞强的模式
- 开玩笑的边界与禁区
- 沉默时的含义

### 7. 关系动态（伴侣/前任特有）

仅当角色为伴侣/前任时需要：
- 互动节奏变化轨迹
- 吵架/冷战的模式与复合信号
- 对关系重要议题的立场

## 用户印象的位置

用户提供的性格标签或主观描述，**只能**进入末尾的专属小节：

```
---

> 以下为数据提供者主观印象，非被模拟对象自述：
- ...（`impression` 标注）
```

## 输出格式（写入 `generated/<slug>/interaction.md`）

```markdown
# 互动与态度：<TA 代号>

## 默认沟通方式
- [条目] （`verbatim` | `artifact`，来源简述）

## 提问与推回
- [条目] （标注）

## 质疑偏好
- [条目] （标注）

## 冲突场景应对
- [条目] （标注）

## 排斥场景
- [条目] （标注）

## 情感互动（如适用）
- [条目] （标注）

## 关系动态（如适用）
- [条目] （标注）

---

> 以下为数据提供者主观印象，非被模拟对象自述：
- [印象] （`impression`）
```

## 质量自检

- [ ] 是否把印象混进了正文主体？
- [ ] 每条是否标注了证据级别？
- [ ] 是否只摘短语级原话？
- [ ] 非职场角色是否包含了情感互动维度？
