# 摄入协议：收集哪些「轨迹」

> 背景：Trace2Skill 证明**多条轨迹并行分析**优于逐条追踪；数字永生需要比单一职场场景更丰富的数据维度。
> 本配方规定材料收集的优先级、元数据、数据源路由与停止条件。

## 数据源路由

根据用户选择的角色模板和可用数据源，自动确定采集策略：

### API 实时拉取（有账号凭证时优先）

| 平台 | 采集器 | 典型场景 |
|------|--------|---------|
| 飞书 | `collectors/feishu.py` | 同事、导师（职场） |
| 钉钉 | `collectors/dingtalk.py` | 同事、导师（职场） |
| Slack | `collectors/slack.py` | 同事、导师（国际团队） |
| Discord | `collectors/discord.py` | 朋友、社群 |
| Telegram | `collectors/telegram.py` | 朋友、伴侣、自己 |
| Email | `collectors/email_collector.py` | 所有角色 |

### 本地数据库读取

| 平台 | 采集器 | 说明 |
|------|--------|------|
| 微信 | `collectors/wechat.py` | 需本地 SQLite 或第三方导出 |
| iMessage | `collectors/imessage.py` | macOS，需 Full Disk Access |

### 归档文件解析

| 平台 | 采集器 | 获取方式 |
|------|--------|---------|
| WhatsApp | `collectors/whatsapp.py` | 应用内「导出聊天」 |
| Twitter/X | `collectors/twitter.py` | 官方数据归档下载 |
| 社交媒体 | `collectors/social_archive.py` | Google Takeout / Facebook 数据下载 |
| 手动 | `collectors/manual.py` | 粘贴/文件上传 |

## 按角色的材料优先级

### 职场角色（同事/导师）

1. **本人主笔的长文**：设计文档、复盘报告、Oncall 记录
2. **决策型短答**：群里对二选一、风险的拍板原话
3. **对他人工作的修改痕迹**：CR 评论、文档批注
4. **一对一指导记录**（导师特有）

### 亲密关系角色（亲人/伴侣/朋友）

1. **私聊记录**：最直接的互动痕迹
2. **群聊中的发言**：展示社交场合的另一面
3. **社交媒体帖子**：公开的自我表达
4. **书信/卡片/语音转文字**

### 自己

1. **自己写的长文**：博客、日记、笔记
2. **聊天记录（自己发出的消息）**
3. **社交媒体发帖**
4. **创作物**：代码、设计稿
5. **问卷自述**：结构化自我回答

### 公众人物（方法论）

对象须有**可追溯的公开出处**（演讲、文章、访谈、官方社交账号等）。

1. **公开文章/博客**
2. **采访/演讲文字版**
3. **社交媒体公开帖子**
4. **新闻报道**（标注 `artifact`）

## 每条材料的元数据标注

| 字段 | 说明 | 示例 |
|------|------|------|
| `channel` | 来源平台 | `feishu` / `wechat` / `imessage` / `twitter` / `paste` |
| `approx_date` | 大致日期 | `2025-Q3`、`2026-01-15` |
| `confidence` | 证据强度 | `verbatim` / `artifact` / `impression` |
| `scope` | 涉及维度 | `procedure` / `interaction` / `memory` / `personality` / `all` |

标注格式（材料开头一行注释）：
```
<!-- channel: wechat | date: 2025-Q3 | confidence: verbatim | scope: interaction -->
```

## 收集时的伦理提示（首轮对话必须告知）

根据角色模板调整伦理声明：

- **所有角色**：产出物是辅助理解工具，不构成冒充授权
- **同事/导师**：限定团队内部对齐与培训用途
- **亲人（已故）**：确认是否有其他家人应知情
- **伴侣/前任**：确认目的是正面回忆而非执念；严格脱敏
- **公众人物**：仅限公开资料；须有可追溯来源
- **朋友**：群聊中他人发言不纳入蒸馏

## 停止条件

按角色所需的维度（参见 `personas/` 对应模板），当每个维度至少有 **3 条带 `verbatim` 或 `artifact` 出处**的条目时，可进入合并阶段。

不足 3 条时，向用户说明缺口并标注「材料不足」。**宁缺毋滥**。
