# 角色模板：公众人物（方法论向）

> 继承 `_base.md` 全部通用维度。本模板**仅基于公开资料**，适用于蒸馏公众人物的公开方法论与表达框架（学者、科技/开源领袖、作家、企业家公开言论等），便于用户构建「多顾问」式知识辅助。

## 激活条件

用户希望「蒸馏某位公众人物的风格」「让 AI 像 XX 一样思考」「学习某位大佬的做事方式」——且能提供**可追溯的公开材料**。

## 核心约束

**本模板禁止模拟私人对话。** 所有材料必须来自公开渠道。

### 对象与来源（硬规则）

- **必须**：对象有明确公开言论/作品/访谈等可引用来源；不编造非公开信息，不冒充本人授权。
- 若用户无法提供任何公开出处或材料明显违法：拒绝生成 Skill，说明原因，建议改用合法公开素材。

## 特有维度（在 _base 基础上新增）

### A. 公开言论与思想

- 核心观点与立场（按主题组织）
- 公开演讲/文章/采访中的经典表述
- 观点随时间的演变轨迹

### B. 公开互动风格

引用 `interaction-mining` 的维度，限定公开场景：
- 社交媒体上的回复风格
- 演讲/访谈中的问答方式
- 面对争议时的公开回应模式

### C. 决策与方法论

引用 `procedural-mining` 的维度（如适用）：
- 公开分享的工作方法/创作流程
- 推荐的工具与资源
- 公开的管理/领导理念

### D. 公开人生故事

引用 `memory-mining` 的维度，限定公开内容：
- 公开分享的人生经历
- 创业/创作历程
- 公开采访中提到的转折点

### E. 性格画像

引用 `personality-mining` 的维度，限定公开表现：
- 公众形象中展现的性格特征
- 明确标注「这是公开形象，可能与私下不同」

## 推荐数据源

1. **公开文章/博客**
2. **社交媒体帖子**：各平台公开帖文、播客链接等（须可追溯到公开账号或官方出处）
3. **采访/播客/演讲**的文字版
4. **公开的代码/项目**（开源作者）
5. **新闻报道**（标注 `artifact`）
6. **Wiki/百科资料**

**禁止数据源**：私人聊天、未公开邮件、泄露文档。

## 伦理特别说明

- 所有内容必须来自**公开渠道**，产出物中需标注来源
- SKILL.md 首行必须声明：「本 Skill 仅基于公开资料生成，不代表本人立场或授权」
- 不得用于生成该人物的虚假发言、伪造观点
- 公众人物也有隐私权，不纳入其公开反对被引用的内容
- 遵循 Fair Use 原则，引用需控制篇幅

## 输出文件结构

```
<slug>/
├── SKILL.md
├── procedure.md      # 公开分享的方法论
├── interaction.md    # 公开互动风格
├── memory.md         # 公开分享的经历
├── personality.md    # 公众形象分析
├── conflicts.md
└── manifest.json
```

## 生成 SKILL.md 的 description 模板

```
"基于公开资料蒸馏 {name} 的思想风格与方法论。仅供学习参考，不代表本人立场或授权。"
```
