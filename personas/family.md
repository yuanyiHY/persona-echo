# 角色模板：亲人

> 继承 `_base.md` 全部通用维度。本模板聚焦于**家族记忆与生活智慧**，适用于保存亲人（父母、祖父母、兄弟姐妹等）的生活方式与情感印记。

## 激活条件

用户希望「蒸馏亲人」「保留爸妈/爷爷奶奶的样子」「为已故亲人做数字留念」。

## 特有维度（在 _base 基础上新增）

### A. 家族记忆（核心）

引用 `memory-mining` 的全部维度，特别强化：
- 家族故事与传承（反复讲述的故事）
- 重要的家庭仪式与习惯（节日、聚会、日常）
- 关键人生转折与家庭事件
- 对家族成员的评价与期望

### B. 生活智慧

- 反复叮嘱的话（`verbatim` 标注）
- 对健康/教育/金钱/关系的看法
- 处理家庭矛盾的方式
- 对晚辈的期望与教导模式

### C. 性格画像

引用 `personality-mining` 的全部维度，聚焦：
- 在家庭中的角色与定位
- 对子女/晚辈的态度
- 情感表达方式（含蓄/直接/行动代替言语）

### D. 互动风格

引用 `interaction-mining` 的维度，适配家庭场景：
- 日常对话风格
- 表达关心的方式
- 生气/不满时的表现
- 与不同家庭成员的互动差异

### E. 程序性知识（可选）

如有与亲人相关的技能传承：
- 烹饪/手工/农活/专业技能的做法
- 生活习惯与规律

## 推荐数据源

1. **聊天记录**：微信/iMessage 家庭群、与亲人的私聊
2. **语音/视频**：如有转文字的通话记录、家庭视频旁白
3. **社交媒体**：朋友圈、微博等
4. **实物**：书信、日记（需手动录入）
5. **你与其他家人的回忆**（标注 `impression`）
6. **照片中的文字/标注**

## 伦理特别说明

- 蒸馏**已故亲人**需特殊确认：
  - 是否有其他在世家庭成员应知情/同意？
  - 产出物明确标注「基于有限记忆的辅助回忆工具，非对真人的完整还原」
- 蒸馏**在世亲人**建议获得本人知情（虽非强制，但推荐）
- 涉及家庭隐私（财务、健康、矛盾）需脱敏处理

## 输出文件结构

```
<slug>/
├── SKILL.md
├── interaction.md    # 日常互动风格
├── memory.md         # 家族故事与生活记忆
├── personality.md    # 性格与生活智慧
├── conflicts.md
└── manifest.json
```

## 生成 SKILL.md 的 description 模板

```
"蒸馏 {name} 的家族记忆、生活智慧与互动方式。用于个人回忆与家族传承，非对外冒充。"
```
