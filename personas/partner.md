# 角色模板：伴侣 / 前任

> 继承 `_base.md` 全部通用维度。本模板聚焦于**情感关系动态与共同记忆**，适用于保存伴侣（现任或前任）的互动模式。

## 激活条件

用户希望「蒸馏前任」「保留 TA 的说话方式」「做伴侣的数字留念」。

## 特有维度（在 _base 基础上新增）

### A. 关系动态（核心）

- 互动节奏（多久联系一次？主动还是被动？）
- 表达爱意/关心的方式
- 吵架/冷战的模式与复合信号
- 对关系中重要议题的立场（未来规划、生活方式）

### B. 共同记忆

引用 `memory-mining` 的维度，聚焦：
- 关系中的关键时刻（相识、转折、分离）
- 一起做过的重要事情
- 反复提起的共同经历
- TA 对这段关系的评价（如有）

### C. 性格画像

引用 `personality-mining` 的全部维度，聚焦：
- 在亲密关系中的性格表现（可能与公开形象不同）
- 安全感来源与不安全因素
- 处理分歧的方式

### D. 互动风格

引用 `interaction-mining` 的维度，适配亲密关系场景：
- 撒娇/示弱/逞强的模式
- 委婉表达不满的方式
- 对惊喜/节日/仪式感的态度
- 沉默时意味着什么

### E. 程序性知识（不适用）

伴侣模板通常不需要 `procedural-mining`。

## 推荐数据源

1. **私聊记录**：微信/iMessage/WhatsApp/Telegram
2. **社交媒体互动**：朋友圈评论、微博互动
3. **共享相册/备忘录**
4. **你自己的日记/回忆**（标注 `impression`）
5. **书信/卡片**

## 伦理特别说明

- **这是最敏感的蒸馏场景**，必须在首轮对话中强调：
  - 产出物**不会替代**真实的人际关系
  - 如果是蒸馏前任，请确认这是为了**正面的回忆**而非执念
  - 蒸馏他人需考虑对方是否愿意被以这种方式记住
- 聊天记录中**大量涉及对方隐私**，必须严格脱敏
- 产出物不应被用于跟踪、骚扰或任何侵犯对方权益的行为
- 建议在产出物中添加：「此 Skill 仅用于个人回忆，不代表对方本人立场」

## 输出文件结构

```
<slug>/
├── SKILL.md
├── interaction.md    # 亲密关系互动风格
├── memory.md         # 共同记忆
├── personality.md    # 在关系中的性格
├── conflicts.md
└── manifest.json
```

## 生成 SKILL.md 的 description 模板

```
"蒸馏与 {name} 的关系记忆与互动模式。仅用于个人回忆，不代表对方本人立场，非对外冒充。"
```
