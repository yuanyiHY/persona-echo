# 角色模板：同事

> 继承 `_base.md` 全部通用维度。本模板聚焦于**职场程序性知识**，适用于保存同事的可复用工作方式。

## 激活条件

用户希望「蒸馏同事」「保留 TA 的工作方式」「把同事变成 Skill」。

## 特有维度（在 _base 基础上新增）

### A. 程序性知识（核心）

引用 `procedural-mining` 的全部维度：
- 工具与环境
- 任务推进模式
- 验收与交付标准
- 领域词典
- Gotchas

### B. 职场互动（核心）

引用 `interaction-mining` 的全部维度：
- 元话语习惯
- 提问与推回
- 质疑偏好
- 冲突处理
- 排斥场景

### C. 性格简要（可选）

仅从 `personality-mining` 中提取与工作相关的部分：
- 职场中的价值观倾向
- 抗压方式

### D. 记忆（不适用）

同事模板**不需要** `memory-mining`。职场关系不需要人生故事。

## 推荐数据源

1. **TA 主笔的长文**：设计文档、复盘报告、Oncall 记录
2. **决策型短答**：群里的拍板原话
3. **对他人工作的修改痕迹**：CR 评论、文档批注
4. **飞书/钉钉/Slack 群聊记录**
5. **你与 TA 的协作印象**（标注 `impression`）

## 输出文件结构

```
<slug>/
├── SKILL.md          # 入口
├── procedure.md      # 程序性知识
├── interaction.md    # 互动风格
├── conflicts.md      # 矛盾裁定
└── manifest.json     # 元数据
```

## 生成 SKILL.md 的 description 模板

```
"模拟 {name} 在 {scope} 场景下的工作方式与沟通边界。用于团队内部对齐与培训，非对外冒充。"
```
