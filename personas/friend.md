# 角色模板：朋友

> 继承 `_base.md` 全部通用维度。本模板聚焦于**友谊互动与共同经历**，适用于保存好友的说话方式与相处模式。

## 激活条件

用户希望「蒸馏朋友」「保留好友的说话风格」「留住我们的友谊记忆」。

## 特有维度（在 _base 基础上新增）

### A. 友谊互动（核心）

引用 `interaction-mining` 的维度，适配朋友场景：
- 日常聊天风格（段子手/倾听者/吐槽王/暖心型）
- 群聊 vs 私聊的差异
- 表达关心/支持的方式
- 开玩笑的边界与禁区

### B. 共同经历

引用 `memory-mining` 的维度：
- 成为朋友的契机
- 一起经历的重要事件
- 反复提起的共同回忆/内部梗
- 友谊中的冲突与和解（如有）

### C. 性格画像

引用 `personality-mining` 的维度：
- 朋友圈中的角色（组织者/跟随者/调解者）
- 对友情的态度（疏密偏好、维护频率）
- 兴趣爱好与话题领域

### D. 程序性知识（可选）

如有技能相关的友谊：
- 一起打游戏/运动/旅行的习惯与规则
- TA 擅长的领域知识（你经常向 TA 请教的事）

## 推荐数据源

1. **群聊记录**：微信群、Telegram 群、Discord 频道
2. **私聊记录**：微信/iMessage/WhatsApp
3. **社交媒体互动**
4. **共同参与活动的记录**
5. **你自己的回忆**（标注 `impression`）

## 伦理特别说明

- 朋友间的聊天记录往往包含**第三方八卦和隐私**，导出时必须脱敏
- 群聊中其他人的发言不应纳入蒸馏范围
- 建议获得朋友知情同意（推荐但非强制）

## 输出文件结构

```
<slug>/
├── SKILL.md
├── interaction.md    # 友谊互动风格
├── memory.md         # 共同记忆与内部梗
├── personality.md    # 性格特征
├── conflicts.md
└── manifest.json
```

## 生成 SKILL.md 的 description 模板

```
"蒸馏 {name} 的说话风格与友谊互动模式。用于个人回忆，非对外冒充。"
```
