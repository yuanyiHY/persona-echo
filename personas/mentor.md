# 角色模板：导师 / Mentor

> 继承 `_base.md` 全部通用维度。本模板聚焦于**教学方式与指导智慧**，适用于保存导师/前辈的指导模式。

## 激活条件

用户希望「蒸馏导师」「保留老师的教导方式」「留住 mentor 的指导风格」。

## 特有维度（在 _base 基础上新增）

### A. 教学模式

- 讲解风格（从具体到抽象 vs 从理论到实践？）
- 反馈方式（直接指出 vs 引导式提问？）
- 对学员犯错的态度与典型回应
- 设定目标与检查进度的习惯
- 推荐的学习资源与路径

### B. 核心教诲

- 反复强调的原则与理念
- 经典语录（`verbatim` 标注）
- TA 讲过的案例/故事，以及其中的教训
- 对行业/领域的独特见解

### C. 程序性知识（次要）

引用 `procedural-mining` 的子集：
- 仅提取与教学内容直接相关的技术/方法
- 不需要完整的工具链记录

### D. 互动风格

引用 `interaction-mining` 的全部维度，特别关注：
- 一对一指导时的沟通模式
- 面对学员疑问的回应方式
- 鼓励/批评的平衡点

### E. 记忆（可选）

从 `memory-mining` 中提取：
- 导师自己分享过的职业经历
- TA 提到的关键转折与反思
- 对你（学员）产生深刻影响的具体事件

## 推荐数据源

1. **导师的课程/讲座笔记**
2. **一对一交流记录**（聊天/邮件）
3. **导师的公开文章/演讲**
4. **你自己的学习笔记**（标注 `impression`）
5. **同门/同学的补充**（标注 `impression`）

## 输出文件结构

```
<slug>/
├── SKILL.md
├── procedure.md      # 教学相关的技术知识
├── interaction.md    # 教学风格与沟通方式
├── memory.md         # 导师分享的经历与故事
├── personality.md    # 教学理念与价值观
├── conflicts.md
└── manifest.json
```

## 生成 SKILL.md 的 description 模板

```
"蒸馏 {name} 的指导方式与核心教诲。用于延续导师的教学智慧与指导风格，辅助学习与回顾。"
```
