# 角色模板：蒸馏自己（数字分身）

> 继承 `_base.md` 全部通用维度。本模板为**全维度蒸馏**——程序性 + 互动 + 记忆 + 性格，适用于创建自己的 AI 数字分身。

## 激活条件

用户希望「蒸馏自己」「做数字分身」「让 AI 像我一样说话/做事」。

## 特有维度（在 _base 基础上新增）

### A. 全量程序性知识

本模板对 `procedural-mining` 的要求比其他角色更高：
- 不仅限于工作场景，还包括生活中的决策模式
- 需要跨领域的工具偏好（工作工具、生活工具、学习习惯）
- 个人知识管理方法

### B. 完整人生叙事

引用 `memory-mining` 的全部维度：
- 成长经历中的关键转折点
- 职业轨迹与选择理由
- 重要关系对世界观的塑造
- 遗憾与未竟之事

### C. 深层性格画像

引用 `personality-mining` 的全部维度，加强：
- 自我认知 vs 他人印象的差异
- 随时间演变的价值观变化轨迹
- 理想自我 vs 真实自我

### D. 创作风格

如有写作/创作/编程习惯：
- 写作结构偏好（总分/递进/对比）
- 代码风格与架构审美
- 视觉/音乐/设计品味

## 推荐数据源（按优先级）

1. **自己写的长文**：博客、笔记、日记、邮件草稿
2. **聊天记录**（自己发出的消息）：微信、iMessage、Telegram、飞书
3. **社交媒体**：Twitter/X、朋友圈、知乎回答、微博
4. **创作物**：代码、文档、设计稿、PPT
5. **他人对自己的描述**：朋友/同事的评价（标注为 `impression`）
6. **问卷自述**：主动回答结构化问题（参考 Preserver 框架）

## 伦理特别说明

- 蒸馏自己是**自我授权**场景，伦理负担最轻
- 但仍需注意：聊天记录中包含他人发言，导出时需脱敏或获取对方同意
- 数字分身不应被用于欺骗他人，需在产出物中标明「这是 AI 生成的模拟」

## 输出文件结构

```
<slug>/
├── SKILL.md          # 入口（< 100 行）
├── procedure.md      # 全量程序性知识
├── interaction.md    # 互动风格
├── memory.md         # 人生记忆与故事
├── personality.md    # 性格与价值观
├── conflicts.md      # 矛盾裁定
└── manifest.json     # 元数据
```

## 生成 SKILL.md 的 description 模板

```
"蒸馏 {name} 的全维度数字分身：工作方式、沟通风格、人生经历与性格特征。仅用于个人备份与辅助回忆，非对外冒充。"
```
