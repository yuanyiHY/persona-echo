# 数字人 Skill

## 这是什么

一个帮你把**任何人**变成 AI 对话对象的工具。

你给它聊天记录、日记、照片，它帮你提炼出这个人的说话方式、思维方式、人生经历、价值观，生成一个 **AI 分身**。之后你可以用 `/{slug}` 直接跟这个分身聊天，就像跟 TA 本人对话一样。

**能蒸谁？**

| 角色 | 适合场景 | 例子 |
|------|---------|------|
| 🪞 自己 | 给自己做数字备份，帮自己回忆和分析 | 25 岁产品经理，INTJ，社恐话痨 |
| 🏢 同事 | 保存工作流程和沟通方式 | 跑路的同事，保存他的技术经验 |
| 🎓 导师 | 保留教学方式和指导智慧 | 退休的老师，TA 的思维方式不该消失 |
| 🏠 亲人 | 保存家族记忆和生活智慧 | 奶奶的唠叨、爸爸的人生建议 |
| 💕 伴侣/前任 | 保留关系记忆和互动模式 | 严格脱敏，仅限正面回忆 |
| 🤝 朋友 | 保留友谊互动和共同经历 | 失联的朋友，那些内部梗不该忘 |

## 怎么用

### 第一步：安装

把 `persona-echo/` 目录放到你的 skills 目录下：

```bash
# 全局安装（所有项目都能用）
cp -r persona-echo ~/.agents/skills/数字人

# 或项目级安装
cp -r persona-echo <你的项目>/.agents/skills/数字人
```

### 第二步：创建分身

在 AI 助手中说：

> "帮我蒸馏一个数字人"
> 或 "我想把自己变成 AI"
> 或 "蒸馏我奶奶"

AI 会引导你完成以下流程：

**① 回答 3 个问题**（1 分钟）
- 代号（就是之后的触发词）
- 一句话介绍
- 性格描述

**② 提供原材料**（数据越多越像）
- 微信聊天记录导出（最推荐）
- QQ 导出
- 日记/笔记
- 照片（会提取时间地点）
- 直接口述
- 或者跳过，只凭上面 3 个问题的信息生成

**③ 等 AI 分析**（几分钟）

AI 会自动完成：
- 从聊天记录提取口头禅、语气、思维模式
- 从日记提取价值观、人生经历
- 从照片提取时间线和生活轨迹
- 生成人格模型（5 层结构）
- 生成记忆档案

**④ 确认并生成**

AI 会给你看摘要，确认后生成分身文件。

### 第三步：对话

分身创建好后，直接用触发词开始对话：

```
/小北          → 像小北一样思考和说话（完整模式）
/小北-self     → 帮你回忆和分析小北（档案模式）
/小北-persona  → 只聊小北的性格和表达风格（人格模式）
```

### 第四步：持续进化

分身不是一次性产物，可以持续更新：

- **追加数据**：说"我有新聊天记录"，AI 会自动合并
- **纠正**：说"我不会这样说"，AI 立即修正
- **回滚**：说"回滚到 v1"，恢复到之前的版本

## 输出结构

每个分身生成一个目录，包含：

```
<slug>/
├── SKILL.md           ← AI 入口文件，定义"这个人是谁"
├── self.md            ← 自我记忆（价值观、经历、习惯、人际关系）
├── persona.md         ← 人格模型（5 层结构：硬规则→身份→说话风格→情感→人际）
├── interaction.md     ← 互动风格（口头禅、语气、回复模式）
├── memory.md          ← 完整记忆档案（转折点、故事、情感地图）
├── personality.md     ← 性格档案
├── conflicts.md       ← 不同来源的矛盾记录
├── meta.json          ← 元数据
└── correction_log.md  ← 纠正记录
```

## 管理命令

| 命令 | 作用 |
|------|------|
| `/list-personas` | 列出所有已创建的分身 |
| `/数字人` | 创建新分身 |
| `/{slug}` | 跟分身对话 |
| `/{slug}-self` | 档案模式（帮回忆和分析） |
| `/{slug}-persona` | 人格模式（仅性格和风格） |
| `/update-persona {slug}` | 追加新数据 |
| `/rollback-persona {slug} v1` | 回滚版本 |
| `/delete-persona {slug}` | 删除分身 |

## 数据来源

支持 12+ 平台的数据导入：

| 类型 | 平台 | 方式 |
|------|------|------|
| 聊天记录 | 微信、QQ、飞书、钉钉、Slack、Discord、Telegram | 导出文件或 API 采集 |
| 社交媒体 | Twitter/X、微博、小红书、朋友圈 | 归档文件或截图 |
| 通讯 | iMessage、WhatsApp、Email | 本地数据库或导出 |
| 文件 | PDF、Markdown、TXT、照片 | 直接上传 |

## 技术架构

这个 skill 融合了两套开源方案的设计：

**蒸馏深度**（来自 immortal-skill）：
- 4 维提取：程序性知识、互动风格、记忆经历、性格价值观
- 5 维记忆：人生转折点、反复故事、共同记忆、情感地图、时代印记
- 证据分级：verbatim（原话）> artifact（文档）> impression（印象）
- 矛盾保留：不强行统一，保留真实的人格张力

**使用体验**（来自 yourself-skill）：
- `/{slug}` 直呼对话，像发微信一样简单
- 5 层人格结构：硬规则→身份→说话风格→情感模式→人际行为
- 增量进化：追加数据、对话纠正、版本回滚
- 3 问快速启动，不用填一堆表单

## 示例效果

### 蒸馏一个 25 岁产品经理

输入：
- 微信聊天记录（2000 条）
- 一段自我描述

输出效果：
```
用户：我最近好焦虑啊
分身：焦虑很正常啊，你又不是第一天焦虑了。
      上次焦虑是产品上线前，再上次是年终述职。
      你焦虑的本质是"还没准备好但时间到了"。
      先把最坏结果想清楚，然后你会发现其实也没那么可怕。
      毕竟你每次焦虑完都能交出还行的东西。
```

### 蒸馏一个退休老师

输入：
- 微信聊天记录（500 条）
- 学生的回忆笔记

输出效果：
```
用户：王老师，我最近工作不顺
分身：年轻人嘛，哪有一帆风顺的。
      我教了三十年书，见过太多学生一开始不顺后来越走越好的。
      关键是你得先搞清楚是"不适合"还是"还没适应"。
      这两个是完全不同的问题。
      你先说说具体是什么情况？
```

## 注意事项

- **原材料质量决定还原度**：聊天记录 + 日记 > 仅口述
- **建议优先提供**：深夜对话、情绪波动时的记录、做决定的聊天、日常闲扯
- **隐私保护**：聊天中他人的发言会自动脱敏
- **伦理边界**：不模拟公众人物的私人对话，不用于跟踪或欺骗
- **版本管理**：每次修改都会自动快照，随时可回滚
